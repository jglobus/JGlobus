Globus SAML Library Change Log

Initial port of OpenSAML 1.1:

1. imported OpenSAML 1.1b

2. changed org.opensaml package to org.globus.opensaml11.saml

3. changed dist jar name to globus-opensaml-1.1.jar

4. imported select SAML2 metadata processing classes from the work done in
   the Shibboleth 1.3 libraries

   changes needed:

   4a - moved package named from edu.internet2.middleware.shibboleth
        to org.globus.opensaml11.md

        removed unecessary subpackages:
        aa/ artifact/ aap/ log/ serviceprovider/ wayf/

   4b - moved shib schemas/ dir to the org.globus.opensaml11.md package
        in order to not conflict with the regular schemas

   4c - To prevent bloat, removed lone xmlbeans Exception reference from:
        org.globus.opensaml11.md.common.provider.ShibbolethTrust:

        public void initialize(Node dom) throws XmlException, ShibbolethConfigurationException {

            }

   4d - added schemas dir copyover to build.xml for
        org.globus.opensaml11.md.schemas

   4e - added schema init methods to org.globus.opensaml11.md.xml.Parser to
        allow for schema dir override

   4f - wrapped byte[] in ByteArrayInputStream for Globus' bounycastle version
        in org.globus.opensaml11.md.common.provider.ShibbolethTrust

   4g - added check for EntitiesDescriptor to XMLMetadata

   4h - modified XMLMetadataProvider to accomodate the various fixes for
        problems found in this thread "Null pointer on Shib 1.3":
        http://www-unix.globus.org/mail_archive/gridshib-beta/2005/10/threads.html#00000

   4i - renamed variables named 'org' to 'organization' in XMLMetadataProvider,
        there were conflicts when moving to the new package name

   4j - added commons-digester.jar and jce-jdk13-125.jar to lib directory


5. Added MetadataReader example code under the toplevel examples directory.
   This will only be built (and executed) by running:

   ant example1

   You must first run: ant dist
